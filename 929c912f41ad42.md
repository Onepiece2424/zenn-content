# 起きていた現象と原因
`terraform plan` コマンドを実行した時、最初の1回目は、すぐに実行されたが、2回目以降実行すると、下記のようにタイムアウトになり、実行完了されませんでした。

```
$ terraform plan

Planning failed. Terraform encountered an error while generating this plan.

╷
│ Error: timeout while waiting for plugin to start
│ 
│ 
╵
```

この現象が起きた原因を調べると、私が使用していた Apple Silicon Mac上で Intel版Terraform（darwin_amd64）を使用していたためです。

私が使用していたMacは、Apple Silicon（M1/M2/M3チップ等）搭載のMacでした。

下記のように、今回の現象を解決する前に使用していたTerraformのバージョンとCPUアーキテクチャを調べると、MacOS用 Intel版（x86_64版）darwin_amd64 であり、Apple Silicon（M1/M2/M3チップ等）を表す darwin_arm64 ではありませんでした。

```
$ terraform -v
Terraform v1.15.5
on darwin_amd64
```

また、さらに深く調べると、Intel版Terraform（darwin_amd64）がApple Silicon搭載Macで動作しない・タイムアウトする主な原因は、アーキテクチャの不一致によるプロセスハングです。Rosetta 2を介したエミュレーション環境では、ネットワーク通信やプロバイダーの起動でデッドロックやタイムアウトが発生しやすくなります。

# 解決方法
いろいろ調べると、解決方法は、使用しているPCとインストールしているHomebrew,TerraformのCPUアーキテクチャを合わせることです（私の場合、HomebrewもApple Silicon Macにする必要がありました）。

解決方法の手順としては、下記の通りです。

```
# Apple Silicon版Homebrewを導入
$ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# PATHを変更
$ echo 'eval "$(/opt/homebrew/bin/brew shellenv)"' >> ~/.bash_profile
$ source ~/.bash_profile

# Homebrewのインストール先を確認
$ which brew　（ /opt/homebrew/bin/brew に変更されていればOK）

# Terraform再インストール
$ brew uninstall terraform
$ brew install hashicorp/tap/terraform

# TerraformのバージョンとCPUアーキテクチャを確認
$ terraform -v
Terraform v1.15.5
on darwin_arm64

```

上記のようにすることで、使用していたTerraformのCPUアーキテクチャをApple Silicon（M1/M2/M3チップ等）用を表す darwin_arm64 に変更することができました。
