### CloudFront の基本用語
CloudFront でよく使用されている基本用語を簡単にまとめてみたので、参考にしていただければ幸いです。

| 用語                                         | 説明                                                                              |
| ------------------------------------------ | ------------------------------------------------------------------------------- |
| **CDN（Content Delivery Network）**          | コンテンツをエンドユーザーに高速・低遅延で配信するための分散ネットワーク。CloudFront はその一種。                          |
| **ディストリビューション（Distribution）**              | CloudFront の配信設定の単位。配信元（オリジン）やキャッシュ設定などを定義。                                     |
| **オリジン（Origin）**                           | CloudFront がコンテンツを取得する元。S3 バケット、EC2、Elastic Load Balancer、外部の HTTP サーバーなどが指定可能。 |
| **エッジロケーション（Edge Location）**               | ユーザーに近い場所に設置されたキャッシュサーバー。リクエストが最も近いエッジロケーションにルーティングされる。                         |
| **キャッシュ**                                  | CloudFront がオリジンから取得したコンテンツを一時的に保存しておく仕組み。次回以降のリクエストに高速で応答できる。                  |
| **TTL（Time To Live）**                      | キャッシュの有効期間。一定時間経過後にキャッシュが無効になり、再度オリジンから取得する。                                    |
| **オブジェクト**                                 | CloudFront を通じて配信するファイル（例：HTML、CSS、画像、動画など）。                                    |
| **ビヘイビア（Behavior）**                        | パスごとに異なるキャッシュポリシーやオリジンを設定できるルール。                                                |
| **オリジングループ（Origin Group）**                 | プライマリとセカンダリのオリジンを指定し、プライマリがダウンした場合にフェイルオーバーする設定。                                |
| **署名付き URL / Cookie（Signed URL / Cookie）** | 特定のユーザーにのみアクセスを許可したいときに使うセキュリティ機能。期限付きアクセスなどが可能。                                |
| **キャッシュポリシー / オリジンリクエストポリシー**              | CloudFront にキャッシュ方法やオリジンへのリクエスト方法を細かく指定する設定。                                    |
| **Invalidation（インバリデーション）**                | キャッシュを明示的に削除して、最新のコンテンツを配信できるようにする操作。                                           |

また、CloudFront の基本用語の使い方と用途例をまとめてみたので、こちらも参考にしていただけると幸いです。

1. ディストリビューション（Distribution）
使い方：CloudFront コンソールまたは AWS CLI で「新しいディストリビューションを作成」する。
用途例：S3 にある静的サイト（HTML/CSS/JS）を世界中に配信したいときに、ディストリビューションを作成して S3 バケットをオリジンとして指定。

2. オリジン（Origin）
使い方：ディストリビューション設定時に「オリジンドメイン名」に S3 バケット名や ELB の DNS 名を入力。
用途例：EC2 上のアプリケーションを高速に配信したい場合、EC2 を裏に持つ ELB をオリジンに設定。

3. エッジロケーション（Edge Location）
使い方：ユーザーが CloudFront の配信ドメインにアクセスすると、自動的に最も近いエッジロケーションから配信される（開発者が明示的に操作することはない）。
用途例：アメリカにある EC2 の動画を日本のユーザーにも高速配信。エッジロケーションが東京にあればそこから配信。

4. キャッシュ
使い方：CloudFront はデフォルトでキャッシュするが、キャッシュポリシーや TTL 設定で調整できる。
用途例：バナー画像や CSS ファイルなど頻繁に変わらないファイルは長めにキャッシュして配信効率を上げる。

5. TTL（Time To Live）
使い方：ビヘイビア設定で Minimum TTL, Maximum TTL, Default TTL を指定。
用途例：ニュースサイトで最新記事の画像は頻繁に更新されるため TTL を短く設定（例：300秒）。

6. オブジェクト
使い方：CloudFront 配信 URLにアクセスすると、S3 またはオリジンから該当オブジェクトが配信される。
用途例：EC サイトの商品画像（image.jpg）を CloudFront 経由で高速配信。

7. ビヘイビア（Behavior）
使い方：ディストリビューションに複数のビヘイビアを追加し、パスに応じた処理を設定（例：/api/* は別オリジン）。
用途例：/images/* はキャッシュ有効、/api/* は常にオリジンへリクエスト（キャッシュ無効）

8. オリジングループ（Origin Group）
使い方：2つ以上のオリジンをグループにし、ヘルスチェック失敗時にフェイルオーバー。
用途例：メインの EC2 が落ちたら、自動的にバックアップの S3 サイトに切り替えてユーザーにサービスを提供。

9. 署名付き URL / Cookie（Signed URL / Cookie）
使い方：CloudFront の署名キーとポリシーを使って、URL を一時的に有効にする署名付きリンクを発行。
用途例：会員限定の動画をダウンロード可能にするが、24時間後に無効化したい → 署名付き URL を発行。

10. キャッシュポリシー / オリジンリクエストポリシー
使い方：CloudFront の「キャッシュポリシー」「オリジンリクエストポリシー」を作成・適用して、Cookie, ヘッダー, クエリ文字列の扱いを定義。
用途例：クエリ文字列ごとに異なるページ（例：?lang=en）をキャッシュ → クエリ文字列を含めるように設定。
ログイン状態で内容が変わるページでは Cookie を含めてキャッシュを制御。

11. Invalidation（インバリデーション）
使い方：CloudFront コンソールまたは CLI で /image.jpg や /* などを指定してインバリデーションを実行。
用途例：誤って古い CSS ファイルをアップロードしてしまった → キャッシュを削除して即時に新しいファイルを配信。
